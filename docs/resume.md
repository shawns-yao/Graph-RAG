**Agentic Graph RAG 智能文档问答系统** Python | Neo4j | LangGraph | FastAPI | KNN | PageRank | CoVe
:::
**2025.01-至今**
::: end

**项目描述**：面向医疗领域复杂文档问答、多跳推理与关系查询场景，设计并实现了一套 Agentic Graph RAG 系统。通过融合向量检索、知识图谱、查询路由与自我纠错机制，缓解传统 RAG 在长上下文、关系断链和检索误召回场景下的准确率与稳定性问题。

- 设计骨架图谱抽取流程，采用 **KNN + PageRank** 对文档块进行权重排序，核心内容进入 LLM 实体关系验证，长尾内容通过轻量级实体对齐挂载到图谱；在本地医疗 benchmark 上，深度抽取 chunk 由 16 个降至 5 个，live LLM prompt token 成本降低 **69.68%**，实体图谱覆盖较 skeleton-only baseline 提升 **4.17 个百分点**
- 设计 **PhraseNode（实体层）+ PassageNode（段落层）** 双层图谱结构，通过 `MENTIONED_IN` 证据关联边支持原文溯源，通过 `RELATED_TO` 实体关系边支持关系遍历；在 30 条关系硬负例上，关系假阳性率由共现 baseline 的 **66.67%** 降至 **33.33%**，20 条正向关系召回率 **95.00%**
- 针对传统单层图谱多跳推理“关系爆炸”问题，引入证据段落约束、低信息实体过滤与关系验证规则；24 条多跳任务中 3 跳图遍历准确率达到 **75.00%**，较 1-hop baseline 提升 **29.17 个百分点**
- 实现基于 **query type（查询类型）** 的动态路由与 vector / BM25 / graph 三路融合检索；在 30 题医疗检索证据可回答性评测中，融合检索达到 **63.33%**，相比最佳单通道 **56.67%** 提升 **6.66 个百分点**
- 基于 **LangGraph** 构建检索自循环与 **CoVe（Chain of Verification，验证链）** 答案核验流程，支持召回质量评估、查询改写、工具切换与证据不足时补充召回；10 条固定 trace eval 中 **40.00%** 的问题自动触发补充召回或工具切换，22 条 atomic claims 均完成证据级验证；本地 `.venv` 下复测核心测试集 **315 passed in 2.69s**
